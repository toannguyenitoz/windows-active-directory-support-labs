# Lab 16 - Service Desk Documentation

## Overview

This lab focuses on writing clear IT Support notes after completing technical work.

## Objectives

- Write a clear ticket summary.
- Record symptoms and impact.
- Document troubleshooting steps.
- Document the resolution.
- Record follow-up actions if required.

## Documentation Structure

Use this structure:

1. Issue summary
2. Affected device
3. Reported symptoms
4. Troubleshooting performed
5. Findings
6. Resolution
7. User confirmation
8. Follow-up action

## Example Note

Issue: Windows 11 client could not access domain resources.

Checks completed: confirmed network connection, checked IP configuration, confirmed DNS server, tested server connectivity and confirmed domain sign-in.

Resolution: corrected client DNS setting and confirmed access was restored.

Follow-up: monitor for recurrence.
